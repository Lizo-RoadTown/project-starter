# Plans

Time-bounded plans for in-flight work. Different from proposals — proposals capture *decisions*, plans capture *execution*.

## Workflow

1. Copy `TEMPLATE.md` to `YYYY-MM-DD-short-name.md`. The date is the day the plan was created.
2. Write the plan. Atomic action items. One person could pick this up and execute.
3. Update the file as work progresses — check off items, add notes, link PRs.
4. When the plan is done, either delete it or rename `archive-YYYY-MM-DD-short-name.md`.

## Relationship to proposals

- Proposal answers "what should we build and why."
- Plan answers "how are we executing it this week."

A proposal can have multiple plans (e.g. "Pillar 1B" the proposal → "2026-05-17-runtime-skeleton-implementation" the plan).
