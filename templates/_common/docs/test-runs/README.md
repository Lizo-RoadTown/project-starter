# Test runs

Friction-surface logs from real end-to-end runs. NOT unit/integration test results — those go in CI. This directory is for *real journey* documentation:

- "I signed up for the first time and here's everything that broke."
- "I tried to walk the onboarding as a new user and here's where I bounced."
- "I ran the install steps from a clean machine and here's what wasn't documented."

Each file: dated `YYYY-MM-DD-short-name.md`. Step-by-step narrative + screenshots when relevant. Reviewing these surfaces UX bugs that automated tests miss.
