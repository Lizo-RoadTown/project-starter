Two variants

Pick the shape that fits.

Both variants share the same foundation, described in the next section. They differ in which extras they layer on top — grouped here by purpose, with full breakdowns on the variants docs page.

UI App

scaffolder flag: --type ui-app

For websites, dashboards, marketing pages — anything with a screen.

  • A visual design rulebook Specific rules for layout, color, animation, and accessibility — plus a contract file your project's PRs cite.
  • First-use onboarding patterns A behavioral-research framework for taking new users from curious to committed.
  • Anthropic's frontend plugin General-purpose assistance for component design and visual decisions.

See exactly what each includes →

Agent App

scaffolder flag: --type agent-app

For LLM apps, AI agents, anything that calls Claude or another model.

  • Agent architecture & orchestration How to structure single-agent or multi-agent systems — handoffs, supervision, parallel work.
  • Memory between sessions Patterns for what your agent remembers and how — vector recall, structured facts, or both.
  • Building on the Claude API Anthropic's official SDK skill plus a portable-identity rule so user data survives LLM swaps.

The shared foundation

Both shapes are built on this.

Whichever shape you picked above, every scaffolded project starts with the same foundation — Claude Code's built-in capabilities, plus skills that fit the task, MCP servers that travel with the project, and auto-memory that persists across sessions. The variant extras layer on top.

Built-in capabilities
Tools every Claude Code session ships with — no install needed. Plan mode lets Claude draft an approach before touching code; todo tracking keeps multi-step work visible; the Explore agent searches your codebase without filling up your conversation; web fetch handles one-off lookups.
built-in
Skills
Specialized playbooks Claude pulls in when a task fits — a checklist for code review, an outline-maker for planning, a UI-design rulebook for visual work, an SDK helper for building on Claude's API. Installed once per computer; your project's CLAUDE.md lists which ones to lean on.
user-installed
MCP servers
Extra tools wired to Claude per project. project-starter pins three: Serena (finds code symbols and references without re-reading whole files), Context7 (looks up live library docs — beats outdated training data), and GitHub (reads issues and PRs without leaving the conversation).
.mcp.json
Auto-memory
Persistent notes Claude writes about you and your work, loaded automatically every future session. Saves who you are, feedback you've given, what's happening in this project, and pointers to external systems. Lives on your computer at ~/.claude.
~/.claude

Now do it · three ways · same output

Scaffold a new project & install its skills.

Pick whichever shell you have. Each path is two commands — first scaffold the folder, then install the skills the project's CLAUDE.md references. Skip the install step and the references in your CLAUDE.md won't resolve.

PowerShell Windows default

Native Windows. Ships preinstalled.

# 1. clone the starter
git clone https://github.com/Lizo-RoadTown/project-starter
cd project-starter

# 2. scaffold a project
./scripts/new-project.ps1 `
  -Type ui-app `
  -Name my-site `
  -Target C:\dev

# 3. install the skills (one-time per computer)
./scripts/install-skills.ps1 -Variant ui-app

Bash Mac · Linux · WSL

Standard POSIX shell.

# 1. clone the starter
git clone https://github.com/Lizo-RoadTown/project-starter
cd project-starter

# 2. scaffold a project
./scripts/new-project.sh \
  --type ui-app \
  --name my-site \
  --target ~/dev

# 3. install the skills (one-time per computer)
./scripts/install-skills.sh --variant ui-app

Docker no shell preference

Reproducible scaffold. Install runs on host.

# 1. build & scaffold via container
docker build -t project-starter .
docker run --rm \
  -v "$(pwd)/out:/out" \
  project-starter \
  --type ui-app --name my-site --target /out

# 2. install skills on your HOST (not in the container —
#    skills live at ~/.claude/skills/ where Claude Code runs)
./scripts/install-skills.sh --variant ui-app

After step 3, open SKILLS.md inside your scaffolded project — it lists the Claude Code marketplace plugins (like frontend-design, claude-api, ralph-loop) that have to be installed from inside a Claude Code session with /plugin install. The install-skills script prints the exact commands to copy-paste. Full breakdown at docs/marketplace & plugins.
