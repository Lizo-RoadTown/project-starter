Three ways · same output

Scaffold a new project

Pick whichever shell you have. All three copy the templates, fill placeholders, git init, and make the first commit.

PowerShell Windows default

Native Windows. Ships preinstalled.

# clone the starter
git clone https://github.com/Lizo-RoadTown/project-starter
cd project-starter

# scaffold
./scripts/new-project.ps1 `
  -Type ui-app `
  -Name my-site `
  -Target C:\dev

Bash Mac · Linux · WSL

Standard POSIX shell.

# clone the starter
git clone https://github.com/Lizo-RoadTown/project-starter
cd project-starter

# scaffold
./scripts/new-project.sh \
  --type ui-app \
  --name my-site \
  --target ~/dev

Docker no shell preference

Reproducible. No local toolchain.

# one-time build
docker build -t project-starter .

# scaffold into ./out
docker run --rm \
  -v "$(pwd)/out:/out" \
  project-starter \
  --type ui-app \
  --name my-site \
  --target /out

Two variants

Pick the shape that fits.

Both share _common — same memory hierarchy, same MCP servers, same skill discipline. Each variant adds the skills and contracts specific to its project shape.
