What is all this?

Four concepts you need before the rest of the docs make sense. None of these are complicated — they each get a few paragraphs and a "what success looks like" check.

What is Claude Code?

Claude Code is an AI coding assistant made by Anthropic. You install it once on your computer, point it at a folder, and then chat with it in plain English. It reads your code, writes new code, runs commands, and explains what it's doing.

Think of it as pair-programming with someone who has read every open-source library and never gets tired. You describe what you want; it proposes changes; you accept or redirect.

It comes in a few flavors:

  • CLI — runs in a terminal window. Most flexible.
  • Desktop app (Mac & Windows) — its own window, no terminal needed. Easiest if you're new.
  • Web app at claude.ai/code — same idea, in a browser.
  • VS Code & JetBrains extensions — Claude lives inside your editor.
i

Beginner recommendation: install the desktop app first. It hides the terminal and gives you a clean chat window. You can switch to the CLI or VS Code extension later when you want more control.

Download: claude.com/claude-code

What "project-starter" has to do with it

When you open a folder in Claude Code, it looks for special files like CLAUDE.md and .mcp.json. These files tell Claude:

  • What this project is — the tech stack, the conventions, the goals.
  • Which extra tools to load — like a code-search tool or live docs lookups.
  • What rules to follow — voice, commit style, what to never do.

Without these files, Claude has to re-figure-out your project every time you open a session. With them, every session starts already knowing how things work.

project-starter is a pre-filled set of these files. Instead of writing them from scratch for every new project, you copy the template, fill in your project name, and you're ready.

What is the terminal?

The terminal is a window where you type commands instead of clicking buttons. Every operating system ships with one. It looks old-fashioned, but it's the most precise way to control your computer.

You'll use the terminal to run two commands during setup: git clone (to download project-starter) and a scaffolder script (to make your new project folder). After that, the desktop Claude Code app handles things for you if you prefer.

How to open the terminal

Windows
Press Win + X then click "Terminal" (or "Windows PowerShell").
Or press Win, type "Terminal", press Enter.
Mac
Press Cmd + Space, type "Terminal", press Return.
Or find it in Applications → Utilities → Terminal.
Linux
Press Ctrl + Alt + T (most distributions).
Or open your launcher and search for "Terminal".

What it looks like when it works

A black or dark window with text. The cursor sits after a short prompt that ends with $ (Mac/Linux) or > (Windows). You type a command, press Enter, and the terminal runs it and shows the output. That's it.

!

Two terminal habits worth learning right away:

  • ↑ recalls the previous command. Saves typing.
  • You can paste with right-click (Windows) or Cmd+V (Mac).

What is git & GitHub?

Git is a tool that tracks every change you ever make to a folder of files. You can rewind to any past state, see who changed what, and merge work from multiple people. It's a separate program from your editor — it works on any folder.

GitHub is a website that stores git folders online. Free for personal projects. Two reasons you want it:

  1. Backup. If your laptop dies, your code is still on GitHub.
  2. Shareability. Anyone with the link can see your project, fork it, suggest changes.

The flow you'll do once per project

  1. git init — start tracking a folder (project-starter does this for you in the scaffolder).
  2. git add & git commit — save a snapshot of the current state.
  3. git push — send your snapshots up to GitHub.

You don't have to learn the command-line version yet

GitHub Desktop is a free app that does all of this with buttons. For your first project, this is the easiest path. The walkthrough shows both options.

i

Downloads:

  • Git itself (required either way): git-scm.com
  • GitHub Desktop (recommended for beginners): desktop.github.com
  • Sign up for a free GitHub account: github.com

What is "scaffolding"?

Scaffolding means creating the starting structure of a project from a template. The name comes from construction — scaffolding is the temporary structure that lets you build the real thing.

In coding, scaffolding a project usually means:

  1. Make a new empty folder.
  2. Copy a bunch of starter files into it (config, folder structure, README).
  3. Fill in the project name and a few details so the files match your project.
  4. Start the project's version history (git init).

Doing this by hand every time is tedious and error-prone. A scaffolder script does it in one command.

What project-starter scaffolds

When you run the scaffolder, your new folder ends up with:

  • CLAUDE.md — the file Claude Code reads at the start of every session.
  • .mcp.json — config for extra tools (semantic code search, live docs).
  • docs/ — folders for architecture proposals, plans, and test logs.
  • .gitignore — tells git which files to never track (like secrets and build output).
  • A first git commit, so your version history starts clean.

You pick a variant when you scaffold — ui-app for websites/dashboards, agent-app for AI agents. The variant decides which extra files and skill recommendations get added on top.

!

You're not locked in. The scaffolded files are just files — edit, rename, or delete anything. The starter gives you a sensible default, not a contract.
