Getting started

A click-by-click walkthrough. Start with nothing installed; end with a working project on GitHub, open in Claude Code. About 20–30 minutes, mostly waiting for downloads.

i

What you'll have at the end: a new folder on your computer called my-first-project, a copy of it on GitHub at github.com/<your-username>/my-first-project, and Claude Code open and ready to help you build in it.

!

If you get stuck on any step, scroll to the Troubleshooting section at the bottom — it covers the most common confusion. Every step also has a "you should see…" so you can check you're on track.

1

Make a GitHub account

GitHub is the website your project will live on. Free. Go to github.com and click Sign up.

You'll need a username (this becomes part of your project URLs, so pick something you're OK with publicly), an email, and a password. Verify your email when GitHub sends you the link.

!

Write down your username. You'll need it in a later step.

2

Install git

Git is the tool that tracks changes to your project. Even if you use the GitHub Desktop app later, git itself still needs to be installed.

Go to git-scm.com and click the download button for your operating system. Run the installer with all the default settings — you don't need to change anything.

Check that it worked

Open a terminal (how?) and run:

git --version

You should see something like git version 2.42.0. If you do, git is installed. If you see "command not found," restart your terminal and try again.

3

Install GitHub Desktop

This is the app that puts your code on GitHub with a button click. Not strictly required — you can do it from the terminal — but it's the easiest path for a first project.

Go to desktop.github.com, download, install, and open. Sign in with the GitHub account you just made.

You should see the GitHub Desktop welcome screen with options like "Clone a Repository" and "Create a New Repository." Leave it for now — we'll come back to it in step 7.

4

Install Claude Code

The AI assistant that will work in your scaffolded project.

Go to claude.com/claude-code. For your first project, pick the desktop app (Mac or Windows) — it's the friendliest entry point. Download, install, open, and sign in with your Anthropic / Claude account (create one if you don't have one).

You should see the Claude Code welcome screen with a chat input. Leave it open. We'll point it at your new project in step 9.

i

If you'd rather use Claude Code in your editor instead of as a standalone app, install the VS Code extension instead. It does the same thing, but inside VS Code.

5

Download project-starter

This is the template you'll scaffold your project from.

Open your terminal and run:

# Move to wherever you want to keep your code — pick a folder you can find easily.
# Example for Windows:
cd C:\Users\YOURNAME\Documents

# Or on Mac/Linux:
cd ~/Documents

# Now clone (= download a copy of) the starter:
git clone https://github.com/Lizo-RoadTown/project-starter

# Step into it:
cd project-starter

You should see a few lines of output ending with something like Resolving deltas: 100% (12/12), done. Then your terminal prompt should now end with project-starter.

6

Scaffold your first project

This runs the script that builds your new project folder.

Pick a name

Use lowercase letters, numbers, and dashes only — no spaces or capitals. Good: my-first-project, weather-app. Bad: My First Project (has spaces and capitals).

Run the right command for your shell

Windows PowerShell
./scripts/new-project.ps1 -Type ui-app -Name my-first-project
Mac / Linux Bash
./scripts/new-project.sh --type ui-app --name my-first-project

ui-app is the right pick for websites, dashboards, and most web projects. If you'll be building an AI agent or LLM app, use agent-app instead.

You should see output ending with: 

Done. Next steps:
  cd my-first-project
  # Open CLAUDE.md and fill in any TBD fields
  # Add a remote and push:
  #   gh repo create my-first-project --private --source=. --push

A new folder called my-first-project now exists next to the project-starter folder. It's already a git repository with one commit.

?

"Cannot run script" on Windows? Run this once, then try step 6 again:

Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

Press Y when asked to confirm. This lets PowerShell run local scripts you've downloaded.

7

Install the skills your project references

Your scaffolded CLAUDE.md references several skills and plugins — things like ui-ux-pro-max (a UI design checklist) and frontend-design (Anthropic's frontend plugin). These aren't installed automatically. If you skip this step, Claude will still work — it'll just tell you when a skill isn't available.

Skills install at the user level (~/.claude/skills/), so this is one-time per computer. Future projects on this machine reuse them.

Run the install-skills script

From the project-starter folder (not your new project folder):

Windows PowerShell
./scripts/install-skills.ps1 -Variant ui-app
Mac / Linux Bash
./scripts/install-skills.sh --variant ui-app

Use agent-app instead of ui-app if that's the variant you scaffolded. Or both if you ever expect to do both.

You should see the script run npm and git commands to install the shell-available pieces (ui-ux-pro-max, design-system, agent-memory-systems), then finish with a printed list of /plugin install … commands.

Finish the install inside Claude Code

The printed /plugin install commands install Anthropic marketplace plugins (frontend-design, claude-api, ralph-loop). They can't run from the shell — they run inside an active Claude Code session.

You'll do this in step 9 once you've opened the project in Claude Code. For now, leave the printed list visible — you'll copy from it.

i

Full reference for the marketplace, the /plugin commands, and exactly what each piece does: Marketplace & plugins. Your project also has a SKILLS.md file at its root that lists everything.

?

"npm: command not found" or "git: command not found"? Install Node.js (nodejs.org) and git (git-scm.com). They're required for the auto-installable parts.

8

Put it on GitHub

The easy way — GitHub Desktop

  1. Open GitHub Desktop.
  2. Menu: File → Add Local Repository (or drag the my-first-project folder into the GitHub Desktop window).
  3. Click Publish repository in the top-right.
  4. Pick a name (the folder name fills in automatically), choose Keep this code private (or uncheck if you want it public), click Publish.

You should see a green confirmation. Your project is now on GitHub at github.com/<your-username>/my-first-project.

The terminal way (optional)

If you've installed the gh command-line tool (it ships with GitHub Desktop on Mac/Windows), you can do the same thing from the terminal: 

cd ../my-first-project
gh repo create my-first-project --private --source=. --push
9

Look at what you got

Open the my-first-project folder in your file browser:

  • Windows: File Explorer → Documents → my-first-project
  • Mac: Finder → Documents → my-first-project

You'll see something like:

my-first-project/
├── CLAUDE.md           # what Claude reads at the start of every session
├── .mcp.json           # config for extra tools Claude can use
├── .gitignore          # list of files git should never track
└── docs/
    ├── plans/          # in-flight plans (proposals + action items)
    ├── proposals/      # architecture decisions, captured BEFORE you code them
    ├── test-runs/      # real end-to-end run logs (UX bugs & gotchas)
    └── UX_CONTRACT.md  # the UI discipline every visual change passes

Open CLAUDE.md in any text editor (Notepad, TextEdit, VS Code). Find any line that says TBD — for example Frontend | TBD — and fill in what you actually want to use. If you don't know yet, leave them and ask Claude in step 9 to help you decide.

10

Open the project in Claude Code & finish the plugin install

In the desktop app

  1. Open Claude Code.
  2. Click Open Folder (or Open Project — the wording varies by version).
  3. Navigate to and select my-first-project.

The first time you open it, Claude will ask permission to use the extra tools listed in .mcp.json (Serena for code search, Context7 for live docs, GitHub for repo data). Approve them — they make Claude much smarter about your project.

You should see Claude's chat panel with an input box at the bottom. It already knows about your project — CLAUDE.md loaded automatically.

In the VS Code extension

  1. Open VS Code.
  2. File → Open Folder → pick my-first-project.
  3. Open the Claude Code sidebar (the icon with the Claude logo).

Finish the plugin install

Remember the /plugin install commands the install-skills script printed in step 7? Paste them into Claude Code's chat input now (one per line — Claude Code runs each as a slash command).

For the UI App variant, that's typically:

/plugin marketplace add anthropics/claude-plugins-official
/plugin install frontend-design

For the Agent App variant:

/plugin marketplace add anthropics/skills
/plugin install claude-api@anthropic-agent-skills

/plugin marketplace add anthropics/claude-plugins-official
/plugin install ralph-loop

You should see Claude confirm each install. Run /plugin list first to skip anything already bundled with your Claude Code install.

!

Verify: ask Claude "Which of the skills referenced in CLAUDE.md do you have installed?" and it'll tell you. Install any it flags as missing.

11

Your first prompt

Type something like: 

Read CLAUDE.md, then suggest what the first feature of this project should
be. I want to build a simple weather dashboard but I'm not sure where to start.

Claude will read your CLAUDE.md, ask one or two clarifying questions, and propose a starting plan. You can accept, redirect, or ask for alternatives — it's a conversation, not a one-shot.

!

Troubleshooting

The most common things that go sideways, and what to do about them.

"command not found" after running git or gh

You need to fully close and reopen the terminal after installing a new command-line tool. New tools don't show up in already-open terminal windows.

"Permission denied" running the scaffolder on Mac/Linux

Run this once, then try the scaffolder again:

chmod +x scripts/new-project.sh

"Cannot run script because running scripts is disabled" on Windows

Run this once in PowerShell (press Y to confirm), then try again:

Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

Claude Code can't find uvx

uvx is needed to run the Serena code-search tool. Install it:

Windows
winget install astral-sh.uv
Mac / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

I want to undo everything

The scaffolded folder is just a folder. Delete it from your file browser to undo step 6. To remove the project from GitHub, go to your repo page on github.com, click Settings, scroll to the bottom, and click Delete this repository. None of this affects project-starter itself — that's still in the folder you cloned in step 5.

Still stuck?

Open an issue at github.com/Lizo-RoadTown/project-starter/issues with what step failed and the exact error message. Include your operating system. Better than guessing.
