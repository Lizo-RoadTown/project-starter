Variants & the shared foundation

A short orientation. The full breakdown of every piece is in the sidebar — under UI App and Agent App — one page per piece.

What's a "skill"?

Most of the variant pieces are skills. A skill is a markdown file (a SKILL.md) plus optional supporting files, stored at ~/.claude/skills/ on your computer or shipped inside a plugin. Each skill tells Claude how to handle a specific kind of task — writing UI, planning, reviewing, calling an API — and which patterns to apply. Installed once per machine, then any project can reach for them.

The scaffolded project doesn't install the skills — it references them in CLAUDE.md so Claude knows to reach for them when relevant. If a skill isn't installed when Claude tries to use it, you'll be told and given the install command.

i

A few pieces aren't skills — they're files in your project (like UX_CONTRACT.md) or design principles baked into the CLAUDE.md template. Each piece's documentation page makes the distinction explicit at the top.

The shared foundation

Both variants ship with this set — it's always there, regardless of which variant you pick. The main page's foundation section covers it in summary; for the developer-flavored breakdown see docs/AGENTS.md.

  • Memory hierarchy — proposals, plans, test-runs, auto-memory.
  • Three MCP servers pinned — Serena, Context7, GitHub.
  • Token discipline rules — semantic search over whole-file reads.
  • Voice rules — describe what is; no marketing language.
  • Commit + PR discipline — small PRs, cite proposals.

The two variants

Pick UI App for websites, dashboards, marketing pages — anything with a screen. Pick Agent App for LLM apps, AI agents, anything that calls Claude or another model. Each adds extras on top of the shared foundation; the sidebar has one page per piece.

How Claude reaches for skills

You don't have to memorize skill names. Claude has the list (from CLAUDE.md) and reaches for them when the task fits. Skills install at the user level (one-time per machine); once installed for one project they're available everywhere.

How to add or remove pieces

Variants aren't sacred. After scaffolding, every piece is just a file you can edit:

  • Add a skill to a variant for all future projects → edit templates/<variant>/CLAUDE.md.extension in this repo.
  • Add a skill to your specific project → edit your project's CLAUDE.md directly.
  • Remove the UX contract → delete docs/UX_CONTRACT.md (or just stop citing it).
  • Add a new variant entirely → see CONTRIBUTING.md for the recipe.
