Glossary

Every technical term used across these docs, in plain English. Ordered alphabetically.

Agent
An AI program that takes goals as input and does work over multiple steps to achieve them, usually using tools. Claude Code is an agent. The agent-app variant is for projects where you're building an agent.
Anthropic
The company that makes Claude (the AI) and Claude Code (the coding assistant).
Auto-memory
Files Claude Code writes to ~/.claude/projects/<your-project>/memory/ as you work. Persists across sessions. Saves user role, feedback, project facts, and references — so future sessions start with context already loaded.
Bash
The default shell on Mac and Linux. The "language" you type in the terminal there. On Windows, the equivalent is PowerShell.
Branch (git)
A parallel line of changes in your git history. main is the default branch. You make a new branch to work on something risky without affecting main, then merge it back when it's working.
CLAUDE.md
The file in your project that Claude Code reads at the start of every session. Contains the project's tech stack, conventions, and discipline. project-starter pre-fills this.
Claude Code
Anthropic's AI coding assistant. Available as a CLI, desktop app, web app, or editor extension. See What is Claude Code?
CLI
"Command-Line Interface." A program you run by typing commands in the terminal, instead of clicking buttons. git, gh, and Claude Code's CLI version are all examples.
Clone
To download a copy of a git repository from somewhere (usually GitHub) onto your computer. git clone <url>.
Commit
A saved snapshot of your project at a moment in time. Every commit has a short message explaining what changed. You commit often; commits are how git tracks history.
Context
Everything Claude has loaded in its working memory for the current conversation. Includes your messages, files Claude has read, and tool results. Bigger context = more capable but also slower and more expensive.
Context7
An MCP server that fetches up-to-date docs for libraries and frameworks. Pinned in project-starter's .mcp.json so Claude always has fresh library docs.
Docker
A tool for running programs in isolated, reproducible "containers." project-starter ships a Docker image so the scaffolder works the same way on every machine without installing PowerShell or Bash.
.gitignore
A file that tells git which files to never track. Used to exclude things like passwords, build output, and editor configs that shouldn't end up on GitHub.
GitHub
A website (owned by Microsoft) that hosts git repositories online. Free for personal projects. Your code lives there as a backup and so others can see / contribute.
GitHub Desktop
A free app from GitHub that gives you buttons for the common git actions — commit, push, pull, branch. Easier than the terminal for beginners.
gh
The GitHub command-line tool. Lets you create repos, open PRs, browse issues — without leaving the terminal. Optional. Installed alongside GitHub Desktop on Mac/Windows.
Git
The tool that tracks changes to a folder of files. Separate from GitHub — git runs locally on your computer, GitHub is just a place to keep git repositories online.
LLM
"Large Language Model." The kind of AI that powers Claude, ChatGPT, etc. project-starter's agent-app variant is for projects that build apps on top of LLMs.
MCP
"Model Context Protocol." A standard for connecting extra tools to AI assistants like Claude Code. An MCP server gives Claude new capabilities — like searching your code, fetching docs, or talking to GitHub.
.mcp.json
A config file in your project that lists which MCP servers should be loaded automatically when Claude Code opens it. project-starter pins three: Serena, Context7, and GitHub.
npx
A tool that runs Node.js packages without permanently installing them. Used here only as an option for the local static server when previewing the docs site.
Plan (project-starter sense)
A dated, time-bounded execution list in docs/plans/. Different from a proposal: proposals decide what to build, plans decide how to ship it this week.
PowerShell
The default shell on Windows. The "language" you type in the terminal there. Equivalent to Bash on Mac/Linux.
PR
"Pull Request." A proposal on GitHub to merge a branch into main. PRs have descriptions, allow code review, and run automated checks before merging.
Proposal (project-starter sense)
An architecture decision captured in docs/proposals/ before code is written. Has a Problem → Decision → Migration path structure. Searchable history of why the project is shaped the way it is.
Push
Send your local commits up to GitHub. git push in the terminal, or the "Push origin" button in GitHub Desktop.
Render
A cloud hosting service. project-starter's demo site is configured to deploy on Render via render.yaml.
Repository (repo)
A folder that git is tracking. Local repos live on your computer; remote repos live on GitHub. "Repo" is short for "repository."
Scaffold
To create a new project's starting structure from a template — make the folder, copy in the files, fill in the project name. See What is scaffolding?
Serena
An MCP server that gives Claude semantic code search powered by language servers. Lets Claude find symbols and references without reading whole files. Pinned in project-starter's .mcp.json.
Shell
The "language" you type in the terminal. PowerShell on Windows, Bash on Mac/Linux. (Technically the shell is the program; the terminal is the window. People use the words interchangeably.)
Skill (Claude Code sense)
A markdown file at ~/.claude/skills/ that tells Claude how to handle a specific kind of task — planning, code review, refactoring, etc. Auto-loaded based on triggers. Installed once per machine.
Stack (tech stack)
The set of technologies a project uses — e.g. "Next.js + FastAPI + Postgres on Render." Filled into CLAUDE.md at scaffold time so Claude knows what to assume.
Template
A pre-built set of files you copy to start a new project from. project-starter ships templates for ui-app and agent-app.
Terminal
The window where you type shell commands. Every OS has one. See What is the terminal?
Token
The unit of text an LLM reads and writes — roughly ¾ of a word. Conversations have token limits and cost money per token. project-starter's CLAUDE.md bakes in "token discipline" so Claude reads efficiently.
UX_CONTRACT.md
A file in the ui-app variant that lists the UX rules every visual change must follow — color semantics, animation budget, accessibility minimums, anti-patterns.
Variant (project-starter sense)
The flavor of project you're scaffolding. ui-app = websites/dashboards. agent-app = LLM agents / Claude API apps. Each variant adds variant-specific skills and contracts on top of the shared _common template.
VS Code
A free code editor from Microsoft. Very popular. Claude Code has a VS Code extension that puts Claude in the sidebar.
