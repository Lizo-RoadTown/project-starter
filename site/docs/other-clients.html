Using project-starter with other Agent Skills clients

project-starter is built primarily for Claude Code, but the open Agent Skills standard means a large subset of what it provides works in Cursor, Gemini CLI, OpenCode, Junie, Goose, Amp, and other compatible clients. This page is honest about what travels and what doesn't.

The split: what's portable, what's Claude-Code-only

Piece Status Notes
The scaffolder scripts universal Just shell scripts that copy files. Agent-agnostic.
Memory hierarchy (proposals/, plans/, test-runs/) universal Just folders + markdown templates. Works in any agent or no agent.
UX_CONTRACT.md universal Plain markdown. Every agent that reads project files can use it.
Portable-identity discipline universal Design principle. Works regardless of agent.
onboarding-psychologist Agent Skills compatible SKILL.md follows agentskills.io spec. Works in every compatible client.
ai-agents-architect Agent Skills compatible SKILL.md follows agentskills.io spec. Works in every compatible client.
agent-memory-systems Agent Skills compatible Standard SKILL.md. Concept (CoALA framework) is agent-agnostic.
ui-ux-pro-max Agent Skills compatible uipro init --ai claude targets Claude specifically, but the underlying skill files install for other clients too.
design-system Agent Skills compatible The pattern is universal. The Triptease implementation is just files; copy into any client's skills dir.
UX_CONTRACT.md (full content) universal A project file. Reference it from whatever your client's "always-load context" mechanism is.
frontend-design Claude Code only Anthropic plugin distributed via Claude Code's marketplace. No equivalent in other clients (yet).
claude-api Claude Code only Anthropic plugin. About building apps with the Claude API, so naturally Claude-specific.
ralph-loop Claude Code only Uses Claude Code's Stop hook system, which is Claude-Code-specific.
agent-orchestrator mixed The patterns are universal. AWS CAO works anywhere; the muratcankoylan skill plugin is Claude Code only.

How to install Agent-Skills-compatible skills in your client

Every client has its own way of registering skills. The skill files themselves (the SKILL.md + supporting files) are the same. Find your client's docs:

Cursor

Cursor uses .cursor/rules/ and a separate skills system. See Cursor's skills docs. Copy SKILL.md files from a marketplace clone into the appropriate Cursor directory.

Gemini CLI

Gemini CLI's skills live in a specific directory loaded at startup. See Gemini CLI skills docs.

OpenCode

OpenCode loads skills from .opencode/ or a global skills directory. See OpenCode skills docs.

Junie

JetBrains' Junie supports Agent Skills directly. See Junie's Agent Skills docs.

Goose, Amp, OpenHands, and others

See the full agentskills.io client showcase — each card links to that client's specific skill-registration docs.

Using Liz's skills outside Claude Code

The two skills authored for project-starter (onboarding-psychologist and ai-agents-architect) ship from Lizo-RoadTown/claude-skills-marketplace. For non-Claude-Code clients:

  1. Clone the marketplace: git clone https://github.com/Lizo-RoadTown/claude-skills-marketplace
  2. The skills are at claude-skills-marketplace/plugins/<name>/skills/<name>/SKILL.md
  3. Copy each SKILL.md + its parent directory into your client's skills directory (or symlink it)
  4. Restart your agent client so it picks up the new skills

What you give up without Claude Code

  • No frontend-design, claude-api, or ralph-loop equivalents. These are Anthropic plugins for Claude Code specifically.
  • No MCP server pinning via .mcp.json. Other clients have their own tool/integration mechanisms.
  • Auto-memory lives somewhere different. The ~/.claude/projects/<name>/memory/ auto-memory system is Claude-Code-specific; other clients have their own persistence approaches.
  • The CLAUDE.md filename convention. Cursor uses .cursorrules; Gemini CLI uses GEMINI.md; etc. Same content, different filenames — adapt the scaffold output accordingly, or symlink.

What you keep without Claude Code

  • Every Agent-Skills-compliant skill (the marked ones above)
  • The full memory hierarchy (proposals, plans, test-runs)
  • The portable-identity discipline
  • The UX contract pattern
  • The scaffolder itself (just renames files at the top)
  • Voice rules, commit/PR discipline, token discipline — all just text in a markdown file

Net: roughly 60-70% of project-starter works without Claude Code. The portable bits also happen to be the most opinionated and most reusable parts.
