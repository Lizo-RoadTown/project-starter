A 99-rule design checklist

The ui-ux-pro-max skill is a design-intelligence engine that bundles 67 UI styles, 161 color palettes, 57 font pairings, 99 UX guidelines, 25 chart types, and 161 reasoning rules. When you ask Claude to design or review a UI, it analyzes the request across five domains in parallel and returns a full design system — pattern, style, colors, typography, effects, anti-patterns to refuse, plus accessibility checks.

What it actually does

When invoked, the skill runs five analyses in parallel — looking at the request through five lenses:

  • Product context — what the user is building (finance dashboard? consumer landing? brutalist portfolio?).
  • Style direction — which of the 67 UI styles fits (glassmorphism, claymorphism, neumorphism, bento grid, brutalism, AI-native, dark mode, etc.).
  • Color palette — chosen from 161 curated palettes, filtered for industry-appropriateness.
  • Landing/layout — section ordering and visual hierarchy.
  • Typography — chosen from 57 font pairings, with a refined body font + distinctive display font.

Reasoning happens via CSV-driven rules — the skill ships ~161 rules that gate which combinations make sense for which use cases. Output is a full design proposal: pattern recommendation, applied style, color tokens, typography choices, effects (animations, shadows), and accessibility pre-flight (contrast ratios, focus states, ARIA labels, touch targets).

When it kicks in

Activates automatically when the prompt touches UI work. Examples:

  • "Design a dashboard for a healthcare app" → triggers
  • "Make this card more readable" → triggers
  • "What font should I pair with Inter?" → triggers
  • "Rename this function" → does NOT trigger
  • "Fix this SQL query" → does NOT trigger

It works in Claude Code as well as Cursor, Windsurf, Antigravity, Codex CLI, Gemini CLI, OpenCode, and Warp.

Anti-patterns it actively refuses

The skill ships a list of patterns it will not produce:

  • AI purple/pink gradients on finance/banking products. Wrong industry signal — users associate them with consumer AI tools, not regulated finance.
  • Bright neon for healthcare or banking. Trust signals demand restraint.
  • Missing focus or hover states. Keyboard navigation must be visible.
  • Emojis as icons. Use SVG; emojis render inconsistently across platforms and don't respect color tokens.
  • Generic body fonts (Arial, system defaults) on marketing sites. Pair a refined body font with a distinctive display font.
  • Harsh, jarring animations. Respect prefers-reduced-motion; keep durations within 150–300ms.

Pre-delivery checklist it runs

Before returning a design, the skill self-audits against:

  • Contrast ratios (≥4.5:1 normal text, ≥3:1 large)
  • Responsive breakpoints (mobile, tablet, desktop)
  • Keyboard navigation paths
  • Focus-state visibility on every interactive element
  • Touch-target minimums (44×44px)
  • ARIA labels on icon-only buttons

How to invoke it

You don't have to. Just describe what you want:

Design a settings panel for a music streaming app. Dark mode default,
should feel premium but not corporate.

The skill will activate automatically and return a structured design proposal. To force it explicitly, mention it by name: 

Use ui-ux-pro-max to redesign this card.

Install

Two steps. First install the CLI globally: 

npm install -g uipro-cli

Then initialize it for Claude Code in your project (or globally): 

uipro init --ai claude

The skill files are written to your Claude Code skills directory. Confirm Claude can see it by asking "is ui-ux-pro-max available?" in a new session.
