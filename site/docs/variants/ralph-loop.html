Iterative agent loops

Anthropic's official plugin for the "Ralph Wiggum" iterative-loop technique. A Stop hook intercepts session-exit attempts and re-feeds the original prompt while preserving all file modifications and git history across iterations. Each iteration sees prior work. Continues until Claude emits a user-defined completion-promise tag.

Type
Plugin (includes a Stop hook, slash commands, scripts)
Maintainer
Anthropic official "Anthropic Verified" plugin
Triggers
Manual — invoked with /ralph-loop
Install
/plugin marketplace add anthropics/claude-plugins-official then /plugin install ralph-loop

What it actually does

Ralph (named after Ralph Wiggum from The Simpsons, because the loop is determined to keep trying) turns a one-shot prompt into a multi-iteration agentic run.

The mechanism is simple:

  1. You run /ralph-loop "<prompt>" with a max iteration count and a completion-promise string.
  2. Claude runs the prompt normally. When it tries to end the session, the Stop hook fires.
  3. The hook checks: did Claude emit the completion promise? If yes, the loop ends. If no, the hook re-feeds the original prompt and lets Claude continue.
  4. File modifications and git state persist across iterations — Claude's next pass sees what the previous pass did.
  5. Loops until completion or until the max iteration count is hit.

A system message tells Claude not to lie about completion just to escape the loop. The honesty constraint matters because the loop can run a long time and the agent could otherwise game its way out.

How to invoke it

/ralph-loop "Implement the user authentication flow with tests"
            --max-iterations 10
            --completion-promise "<promise>COMPLETE</promise>"

Cancel an in-progress loop: 

/cancel-ralph

When it's the right tool

Use ralph-loop for:

  • Greenfield TDD. You have a spec, you want Claude to write the test, then code, then make the test pass, then move on. Each iteration completes a unit.
  • Tasks with automated verification. The loop knows when to stop because tests pass.
  • Clear success criteria. You can articulate what "done" looks like in a promise string.
  • Phased work with checkpoints. Each iteration completes one phase.

When it's the wrong tool

Don't use ralph-loop for:

  • Vague success criteria. "Improve the codebase" has no completion signal. The loop runs until the iteration cap, possibly churning.
  • Complex tasks without phasing. If "done" requires understanding what's in iteration 7 from what happened in iteration 2, the loop's stateless re-feed will lose threads.
  • No iteration limit. Always set --max-iterations. The cost otherwise is unbounded.
  • Tasks where you want to review each step. The loop runs without checking in. Use plan-mode + manual review for those.

The Stop-hook mechanism

Stop hooks fire when Claude attempts to end a session. They can either allow the stop (normal session ends) or block it (Claude continues). Ralph's hook:

  1. Reads the transcript looking for the completion promise.
  2. If found → allows the stop. Loop ends.
  3. If not found → checks iteration count. If under cap, blocks the stop and re-injects the original prompt. Claude continues.
  4. If iteration cap is hit → allows the stop with a warning. Loop ends incomplete.

Plugin structure (for the curious)

ralph-loop/
├── .claude-plugin/      # plugin manifest
├── commands/             # /ralph-loop and /cancel-ralph
├── hooks/                # the Stop hook
│   └── hooks.json
├── scripts/              # helper shell scripts
├── README.md
└── LICENSE
!

Windows note: if hooks fail with "bash not found," set the Git Bash path explicitly in hooks/hooks.json. Default WSL resolution can resolve to the wrong shell on some Windows setups.

How it relates to the other Agent App pieces

  • ai-agents-architect decides whether your app needs an iterative loop at all (vs. one-shot or supervisor/worker).
  • agent-orchestrator is for coordinating multiple agents. Ralph is for getting one agent (or session) to iterate.
  • Ralph pairs naturally with the built-in /loop harness skill for self-paced multi-turn execution.

/plugin marketplace add anthropics/claude-plugins-official
