Anthropic SDK helper

Anthropic's official skill for building apps that call the Claude API. Provides Claude with up-to-date reference for the Messages API and Managed Agents (beta), across 8 languages. Covers prompt caching, tool use, streaming, batching, and model migration. Auto-loads when your code imports the SDK.

Type
Skill (skill suite with subcommands)
Maintainer
Anthropic official Lives in anthropics/skills repo
Triggers
Imports of anthropic or @anthropic-ai/sdk; Claude API questions
Install
Bundled with Claude Code; standalone via /plugin install claude-api@anthropic-agent-skills

What it covers

The skill provides Claude with current, authoritative reference material for the entire Anthropic SDK surface. It's not a tutorial — it's a fast lookup for whatever's relevant to the code you're writing.

Messages API (8 languages)

Quick-start, request/response shape, message roles, system prompts. Covers:

  • Python (anthropic)
  • TypeScript / JavaScript (@anthropic-ai/sdk)
  • Java
  • Go
  • Ruby
  • C# / .NET
  • PHP
  • cURL (for any other language)

Managed Agents (beta, 7 languages)

Anthropic's hosted-agent runtime. Same languages as Messages API minus C#. Includes:

  • The Agent-once / Session-every-run pattern.
  • Required beta header: managed-agents-2026-04-01.
  • Session management and persistence.

Tool use

Letting Claude call your functions. Covers tool schemas, response handling, the new beta tool runner. Documents the iteration pattern where Claude calls a tool → your code executes it → result goes back to Claude → next turn.

Streaming

Show output as Claude generates it instead of waiting for the full response. Covers SSE event handling, partial-message accumulation, cancellation.

Batching

Submit many requests at once for 50% cost reduction. Async processing with a results-fetch step. Covers batch size limits, latency expectations, and when batching is worth it.

Prompt caching

Critical for cost efficiency. The skill covers:

  • Prefix stability — cache hits require byte-identical prefixes. The skill teaches what counts as a prefix change.
  • Breakpoint placement — where to put cache breakpoints to maximize hit rate.
  • Silent-invalidator audit — common things that quietly invalidate the cache (timestamps, request IDs, tool ordering).

Model migration

When Anthropic releases a new model, the skill helps you migrate. Specifically:

  • Replaces deprecated model constants (e.g. Model.CLAUDE_OPUS_4_6 → Model.CLAUDE_OPUS_4_7).
  • Removes parameters no longer accepted (temperature, top_p, top_k for Opus 4.7).
  • Converts thinking: {budget_tokens} → thinking: {type: "adaptive"}.
  • Strips beta headers that have gone GA (e.g. effort-2025-11-24, interleaved-thinking-2025-05-14).

Run it explicitly with /claude-api migrate.

Structured outputs

Modern alternative to prefilling. The skill teaches when to use structured output schemas vs. JSON-mode prefill.

Effort calibration

For models that support effort levels (like Opus 4.7), the skill recommends xhigh for coding and agentic tasks. Lower effort for simpler tasks to save tokens.

When it kicks in

Automatic activation:

  • Your code imports anthropic (Python) or @anthropic-ai/sdk (TS/JS).
  • You ask to build, debug, or tune a Claude API feature.
  • You mention model migration, caching, tool use, streaming, or batching.

Does NOT activate for:

  • OpenAI SDK code (openai import).
  • Other LLM provider SDKs.
  • Provider-neutral wrapper code.
  • General programming or ML questions.

Manual invocation:

/claude-api                            # general help
/claude-api migrate                    # model migration
/claude-api managed-agents-onboard     # start a Managed Agents project

Install

If you have Claude Code: bundled — no install. Confirm: 

/plugin list | grep claude-api

Standalone install: 

# Via skills CLI:
npx skills add https://github.com/anthropics/skills --skill claude-api

# Or via the plugin marketplace:
/plugin marketplace add anthropics/skills
/plugin install claude-api@anthropic-agent-skills

Why it's pinned in the Agent App variant

Agent apps almost universally call an LLM API. If you're building on Claude, having Claude write that code is much smoother when Claude itself knows the current API surface. The auto-load means you usually don't have to ask for it — it activates the moment your code touches the SDK.
