The Claude Code marketplace & plugins

Most of the extras the templates reference are plugins or skills that live in a marketplace. This page explains how the marketplace works, how to install plugins, and exactly which ones this project needs.

i

Quick install: run ./scripts/install-skills.ps1 -Variant ui-app (or ./scripts/install-skills.sh --variant ui-app) from the project-starter root. That handles the shell-installable parts and prints the Claude-Code-only commands for you to copy-paste.

What's a plugin?

A plugin is a bundle of Claude Code extensions — skills, slash commands, hooks, and scripts — packaged together. You install a plugin once per computer; every project on that computer can then use it.

Plugins live in marketplaces — GitHub repos that publish a catalog. Anthropic runs the official marketplaces; community maintainers publish their own.

A skill is one of the things a plugin can contain — a SKILL.md file that tells Claude how to handle a specific kind of task. Some skills ship standalone (not inside a plugin); for those, you install via npm or git clone instead of the marketplace.

The three marketplaces this project uses

1. anthropics/claude-plugins-official

Anthropic's official verified plugin marketplace. Contains frontend-design, ralph-loop, and several others. Most of these come bundled with Claude Code when you install it — but if you installed Claude Code via a method that didn't include them, this is where they come from. 

/plugin marketplace add anthropics/claude-plugins-official
/plugin list                            # see what's available
/plugin install <name>                  # install one

2. anthropics/skills

Anthropic's official skills marketplace — distinct from the plugins marketplace. Contains claude-api and other standalone skills that don't need plugin packaging. 

/plugin marketplace add anthropics/skills
/plugin install claude-api@anthropic-agent-skills

3. Lizo-RoadTown/claude-skills-marketplace

Liz Osborn's own marketplace, the home for skills authored for project-starter's variant templates. Currently ships three plugins:

  • onboarding-psychologist — IDENTITY-TO-HABIT arc for first-time-user flows. Grounded in BJ Fogg + Nir Eyal + James Clear.
  • ai-agents-architect — decision framework for autonomous-agent architecture (autonomy spectrum, ReAct vs Plan-and-Execute vs Tree-of-Thoughts, single-vs-multi-agent, when to introduce an orchestrator).
  • make-skills-discipline — auto-injecting Claude Code plugin that enforces PROBE-first behavior, file:line citation, dev-vs-runtime distinction, and friction-as-memory writing via hooks. Recommended for every Make_Skills or project-starter-scaffolded repo. Recommended in project-starter v0.4.0; required in v0.4.1 after smoke-test.
/plugin marketplace add Lizo-RoadTown/claude-skills-marketplace
/plugin install onboarding-psychologist@lizo-skills
/plugin install ai-agents-architect@lizo-skills
/plugin install make-skills-discipline@lizo-skills

Source: github.com/Lizo-RoadTown/claude-skills-marketplace. The two skill plugins are agentskills.io-compliant, so they also work in Cursor, Gemini CLI, OpenCode, and other compatible clients. The discipline plugin is Claude Code-specific (it relies on Claude Code's hook system).

4. Other community marketplaces

Anyone can publish a marketplace. Examples this project references:

  • muratcankoylan/Agent-Skills-for-Context-Engineering — contains multi-agent-patterns.
  • nextlevelbuilder/ui-ux-pro-max-skill — distributed via npm rather than as a marketplace plugin (use npm install -g uipro-cli).
  • sickn33/antigravity-awesome-skills — distributed as a git repo to clone (contains agent-memory-systems).

The /plugin command — invoke from inside Claude Code

All /plugin commands run inside an active Claude Code session, not in your terminal. Open Claude Code, then type these into the chat input:

  • /plugin marketplace add <repo> — register a marketplace by GitHub repo
  • /plugin marketplace list — show registered marketplaces
  • /plugin install <name> — install a plugin or skill
  • /plugin install <name>@<marketplace> — install from a specific marketplace
  • /plugin list — show installed plugins
  • /plugin remove <name> — uninstall
  • /plugin update — pull latest versions

Why these can't be done from a shell: plugins integrate with Claude Code's settings, skill registry, and hook system at install time. The shell doesn't have that integration; Claude Code itself does.

Standalone skills (not in any marketplace)

Some skills are just SKILL.md files in a GitHub repo. To install them, clone or copy the SKILL.md into your ~/.claude/skills/ directory: 

# Mac/Linux
cd ~/.claude/skills
git clone https://github.com/<user>/<repo> <skill-name>

# Windows PowerShell
cd $env:USERPROFILE\.claude\skills
git clone https://github.com/<user>/<repo> <skill-name>

For some skills (like ui-ux-pro-max) you install a CLI tool that knows how to set the skill up correctly: 

npm install -g uipro-cli
uipro init --ai claude

The install workflow, end to end

  1. Scaffold your project (new-project.ps1 / new-project.sh).
  2. Run the install-skills script for your variant: 
    # Windows PowerShell
./scripts/install-skills.ps1 -Variant ui-app

# Mac/Linux Bash
./scripts/install-skills.sh --variant ui-app

# Add --dry-run to see what would happen first
  3. Open Claude Code in your scaffolded project.
  4. Run /plugin list to see what's already bundled.
  5. Run the /plugin install commands the script printed (only the ones not already present).
  6. Ask Claude "Which of the skills referenced in CLAUDE.md do you have installed?" to verify.

Troubleshooting

"Plugin not found" when running /plugin install

Make sure you ran /plugin marketplace add <repo> first for that plugin's source marketplace. /plugin install looks only in marketplaces you've registered.

"Already exists" from install-skills.ps1

Safe to ignore. The script checks for existing installs and skips them. Re-run with the same args anytime — it's idempotent.

npm or git not found

Install Node.js (includes npm) and git. They're prerequisites for the install-skills script.

The plugin installed but Claude doesn't seem to use it

Open a fresh Claude Code session in your project. Plugin registration reloads at session start. Ask Claude "is <skill-name> available?" to verify it's loaded.
